# 🧩 README — СИСТЕМА v2 (MLM / AI / CRM)

## 1. Назначение проекта

**СИСТЕМА v2** — это CRM с ИИ-агентом для MLM / партнёрского бизнеса,  
созданная для автоматизации до 80% рутинных действий партнёра.

Проект ориентирован:
- не на программистов,
- а на реальных партнёров бизнеса,
- с упором на дупликацию, масштабирование и контроль.

❗ ВАЖНО:  
Система **не гарантирует доход**, не нарушает политику MLM и не ведёт агрессивный маркетинг.

---

## 2. Технологический стек

### Frontend
- React + TypeScript
- Vite
- Tailwind CSS
- shadcn/ui
- Lucide Icons

### Backend
- Supabase (PostgreSQL + Auth + RLS)
- SQL schema: **61 таблица (единый файл)**

### AI / Automation
- n8n — оркестратор workflow
- OpenAI / LLM — логика ИИ-агента
- Supabase DB — источник истины

### Dev tools
- Cursor (основная среда разработки)
- Git (монорепозиторий)

---

## 3. Архитектура проекта (высокий уровень)

UI (React)
↓
Supabase Client
↓
Postgres (61 tables, RLS)
↓
n8n Workflows
↓
AI Agent (logic-first, rule-based)

ИИ **не встроен в UI напрямую**, он работает через события и статусы.

---

## 4. Пользовательские роли

### Partner (основная)
- работает со своими лидами и контактами
- управляет задачами
- разрешает / запрещает ИИ действия

### Owner / Admin
- видит всю структуру
- не имеет доступа к чужим контактам
- управляет ролями и правилами

### AI Agent
- не автономен
- работает строго по статусам
- все действия логирует

## 9. Работа с Supabase

### Auth
- используется `auth.users`
- таблицы `users` в public **НЕТ и не должно быть**

### Связь
- все бизнес-данные через `partners`
- связь user → partner обязательна

### RLS
- включён на всех таблицах
- партнёр видит ТОЛЬКО свои строки
- Owner не видит contacts-домен

❗ Если данные «не вставляются» — в 90% случаев это RLS или partner_id.

---

## 10. ИИ-агент: правила для разработчика

ИИ:
- не пишет без разрешения
- не меняет данные без логирования
- не придумывает тексты (использует templates)
- не действует вне workflow

ИИ — это **исполнитель**, не архитектор.

---

## 11. n8n (логика, не код)

Типовые workflow:
- New Lead → Qualification → Suggest Action
- Status changed → AI decision
- Timer → Follow-up
- AI message → Log → Task

Каждый workflow:
- начинается событием
- заканчивается логом

## 13. Главный принцип проекта

> Система должна помогать партнёру зарабатывать,
> а не превращать его в администратора CRM.

Если решение:
- сложно объяснить,
- сложно дуплицировать,
- требует обучения программиста,

→ оно не подходит.
