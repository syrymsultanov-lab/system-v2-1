# 🚀 ЗАПУСК SYSTEM V2 — ИНСТРУКЦИЯ "ТАНЦЫ С БУБНОМ"
### Выполнять КАЖДЫЙ РАЗ перед началом работы с системой!

---

## ШАГ 1: Запускаем Docker Desktop

1. Найди иконку **Docker Desktop** на рабочем столе или в меню Пуск
2. Дважды кликни — запусти
3. Жди пока в трее (правый нижний угол) иконка Docker станет **зелёной**
4. Это означает что Docker Engine запущен ✅

> ⏱ Обычно занимает 30-60 секунд

---

## ШАГ 2: Запускаем n8n через Docker

1. Открой **PowerShell** или **CMD** (Win+R → cmd → Enter)
2. Введи команду:

```
docker start n8n
```

3. Подожди 10-15 секунд
4. Открой браузер и перейди по адресу:

```
http://localhost:5678
```

5. Должен открыться n8n с твоими воркфлоу ✅

> ❗ Если n8n не открывается — значит контейнер называется иначе.
> Проверь командой: `docker ps -a`
> Найди контейнер с n8n и используй его имя вместо "n8n"

---

## ШАГ 3: Запускаем ngrok

1. Открой **новое окно** PowerShell или CMD
2. Введи команду:

```
ngrok http 5678
```

3. Ngrok запустится и покажет строку вида:

```
Forwarding   https://xxxx-xxxx-xxxx.ngrok-free.app -> http://localhost:5678
```

4. **Скопируй этот https URL** — он нужен на следующем шаге ✅

> ⚠️ ВАЖНО: Каждый раз при перезапуске ngrok URL меняется!
> Бесплатный план ngrok даёт новый URL при каждом запуске.

---

## ШАГ 4: Обновляем Telegram Webhook

> Нужно выполнять ТОЛЬКО если ngrok URL изменился!

Открой браузер и вставь эту ссылку (замени значения):

```
https://api.telegram.org/bot<ВАШ_ТОКЕН>/setWebhook?url=<NGROK_URL>/webhook/telegram
```

**Пример:**
```
https://api.telegram.org/bot7843521234:AAF.../setWebhook?url=https://xxxx.ngrok-free.app/webhook/telegram
```

Должен прийти ответ:
```json
{"ok":true,"result":true,"description":"Webhook was set"}
```

✅ Теперь Telegram знает куда слать сообщения!

---

## ШАГ 5: Обновляем URL в Workflow 1 (если ngrok URL изменился)

1. Открой n8n → **Workflow 1: Webhook Receiver**
2. Кликни на ноду **"HTTP Request"** (последняя нода — Send to AI Engine)
3. Обнови поле **URL**:

```
https://<НОВЫЙ_NGROK_URL>/webhook/ai-processing
```

4. Нажми **Save** (Ctrl+S)

---

## ШАГ 6: Проверяем что всё работает

1. Открой n8n → **Workflow 2: AI Processing Engine**
2. Убедись что статус **"Published"** (зелёный) в правом верхнем углу
3. Нажми кнопку **"Execute workflow"** — он перейдёт в режим **"Waiting for trigger event"**
4. Открой **Workflow 1** — убедись что он тоже **Published** ✅

---

## ШАГ 7: Тестируем систему

1. Открой Telegram
2. Найди своего бота
3. Отправь сообщение: **"Привет"**
4. В n8n должны загореться зелёные ноды
5. Бот должен ответить через 5-10 секунд ✅

---

## 🔧 ЧАСТЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

| Проблема | Решение |
|----------|---------|
| n8n не открывается | Проверь что Docker запущен: `docker ps` |
| Бот не отвечает | Обнови Telegram webhook (Шаг 4) |
| ngrok выдаёт ошибку | Перезапусти: закрой окно, запусти заново |
| Workflow не запускается | Проверь статус Published в обоих воркфлоу |
| Ошибка в нодах | Открой вкладку Executions и посмотри где красный |
