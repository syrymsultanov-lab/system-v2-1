### 3. ✅ Добавление гибких метрик через JSONB

**Создан файл:** `database/migrations/add_custom_metrics.sql`

**Что добавлено:**
- Поле `custom_metrics` (JSONB) в `partner_metrics_daily`
- Поле `custom_metrics` (JSONB) в `partner_metrics_monthly`
- GIN индексы для быстрых JSONB запросов
- Комментарии с примерами использования

**Преимущества:**
- ✅ Клиент может хранить любые метрики (PV/GV для MLM, sales volume для продаж, и т.д.)
- ✅ Не нужно менять схему для каждой компании
- ✅ Полная гибкость и независимость от конкретных брендов

**Примеры использования:**
```json
// MLM
{"pv": 500, "gv": 2000, "rank": "gold"}

// Продажи
{"sales_volume": 10000, "conversion_rate": 0.15}

// Рекрутинг
{"candidates": 50, "interviews": 10, "hires": 3}
```

**Как применить:**
```bash
# Выполните миграцию в Supabase SQL Editor
psql -f database/migrations/add_custom_metrics.sql
# или скопируйте содержимое в Supabase SQL Editor
```
