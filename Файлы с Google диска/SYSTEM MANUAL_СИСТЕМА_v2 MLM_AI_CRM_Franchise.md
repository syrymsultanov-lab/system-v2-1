Этот файл — **единый источник правды** о проекте «СИСТЕМА v2».

Он используется для:
- загрузки контекста в новую ИИ-беседу;

---

## 1. Что такое СИСТЕМА v2

**СИСТЕМА v2** — это:
- CRM + AI-агент
- для MLM / партнёрского бизнеса
- с фокусом на дупликацию, масштабирование и автоматизацию
- без гарантий дохода (этика MLM)

СИСТЕМА создаётся **не для программиста**, а для **партнёра**, который помогает строить бизнес (в т.ч. для Сайры).

ИИ выполняет до **80% рутинной работы**:
- первичный контакт
- сопровождение
- напоминания
- рекомендации
- аналитику

---

## 2. Роли в СИСТЕМЕ

### 2.1 Партнёр
- основной пользователь
- работает с лидами, контактами, задачами
- может передавать работу ИИ

### 2.2 ИИ-агент
- НЕ автономный «болтун»
- действует по правилам и статусам
- использует шаблоны
- пишет только там, где разрешено
- логирует ВСЕ действия

### 2.3 Owner / Администратор
- видит всё, кроме чужих контактов
- управляет структурой и политиками
- не вмешивается в личные контакты партнёров

---

## 3. Основные сущности (логика)

### 3.1 Лид
**Лид** — человек, пришедший:
- из формы
- из лендинга
- из мессенджера
- из ИИ-диалога
- из ручного ввода

Лид — это **вход в систему**.

Жизненный цикл:
`new → in_work → handed_to_ai → converted / rejected / archived`

---

### 3.2 Контакт
**Контакт** — это:
- личный человек партнёра
- импорт (телефон, CSV, VCF)
- или сконвертированный лид

Контакты:
- НИКОГДА не видны другим партнёрам
- Owner НЕ имеет к ним доступа
- могут быть переданы ИИ только по флагу

---

### 3.3 Задача
Используется для:
- напоминаний
- follow-up
- контроля действий ИИ и партнёра

## 5. Назначение ключевых вкладок

### 5.1 Лиды
- единая точка входа
- квалификация
- передача ИИ
- контроль статусов
- аналитика

### 5.2 Контакты
- личная база партнёра
- статусы: hot / warm / cold / do_not_contact
- источник: импорт / лид / ручной ввод
- ИИ работает ТОЛЬКО с разрешёнными

### 5.3 История (MVP)
- журнал событий
- кто, что, когда
- действия ИИ и партнёра
- только чтение

### 5.4 Обучение (MVP)
- модули и уроки
- отметка прохождения
- без тестов
- база для дупликации

### 5.5 Шаблоны (MVP)
- тексты сообщений
- каналы: WhatsApp / Telegram / Email
- используются партнёром и ИИ
- ИИ НЕ придумывает тексты сам

## 7. ИИ-АГЕНТ: философия и правила

### 7.1 Что ИИ МОЖЕТ
- читать статусы
- рекомендовать действия
- начинать диалог (если разрешено)
- писать по шаблонам
- создавать задачи
- логировать действия

### 7.2 Что ИИ НЕ МОЖЕТ
- писать без разрешения
- менять данные молча
- нарушать RLS
- придумывать юридические обещания
- гарантировать доход

---

## 8. ИИ и workflow (n8n / logic-first)

ИИ-агент работает через workflow:
- вход: событие (новый лид, статус, таймер)
- проверка условий
- действие (сообщение / задача / рекомендация)
- логирование

ИИ **не монолит**, а набор сценариев.

---

## 9. Технические договорённости

- Разработка ведётся в Cursor
- Supabase — основной backend
- n8n — оркестратор ИИ
- ChatGPT используется как архитектор и логический ИИ

⚠️ Слова «FINAL» и «окончательный» **не используются**.
Используются:
- рабочая версия
- заморозка
- патч

---

## 10. Главный принцип проекта

> СИСТЕМА должна помогать человеку строить бизнес,  
> а не заставлять его разбираться в технологиях.

Если решение:
- усложняет
- требует объяснений
- ломает дупликацию  

→ значит, оно НЕ подходит.
