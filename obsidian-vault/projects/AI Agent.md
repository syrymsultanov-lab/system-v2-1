# AI Agent

## Статус: Отложен до стабилизации landing + dashboard

## Роль AI
- **Исполнитель**, не архитектор
- Работает строго по статусам и шаблонам
- Все действия логируются в `events_log`
- Не пишет без разрешения партнёра

## Что AI делает
- Квалификация лидов через WhatsApp/Telegram/Email
- Сортировка: hot / warm / cold
- Доставка презентации
- Follow-up по расписанию
- Создание задач для партнёра

## Что AI НЕ делает
- Не принимает решения
- Не придумывает тексты (только шаблоны)
- Не меняет данные без логирования
- Не гарантирует доход

## Условия работы
- `selected_for_ai = true` на лиде/контакте
- Партнёр разрешил AI в настройках
- Лимиты не превышены

## Таблицы
- `ai_jobs` — задачи
- `templates` — шаблоны сообщений
- `events_log` — логи действий

## UX-решения для кнопок в модалке лида
*Зафиксировано 2026-04-18*

### «🤖 Передать AI-агенту» — полуавтомат (не полный автомат)
Flow:
1. Партнёр жмёт → insert в `ai_jobs(lead_id, type='draft', status='pending')`
2. n8n workflow подхватывает → LLM формирует **черновик** первого сообщения по шаблону из `templates` + контексту лида (имя, город, мессенджер)
3. Черновик возвращается в UI (уведомление партнёру) → партнёр **редактирует и подтверждает**
4. После подтверждения — n8n отправляет через WABA/Telegram API → запись в `lead_messages`
5. Ответ лида триггерит следующий раунд черновика → снова на одобрение
6. Статус `new → contacted → qualified / rejected` меняется автоматически, но отправки — только с одобрения человека

**Почему полуавтомат, а не полный автомат:**
- WABA банит за нарушения Meta policy
- LLM может случайно пообещать гарантированный доход → нарушение InCruises policy
- На первых итерациях нужен человеческий надзор; полный автомат — отдельный этап зрелости

### «💬 Открыть переписку» — отдельная страница `lead.html?id=X`
Почему страница, а не split-view в модалке:
- Длинная переписка в модалке неудобна, особенно на мобилке
- Deep-link из push-уведомлений и задач (`?id=X` — single source of truth)
- Нативный back-button работает

Страница `lead.html?id=X`:
- Шапка: данные лида (те же что в модалке) + действия (смена статуса, AI)
- Timeline из `lead_messages` (от партнёра / AI / ответы лида)
- Input снизу: выбор канала (WhatsApp/Telegram/Email) + textarea + кнопка «Отправить»
- Отправка: insert в `lead_messages` → n8n подхватывает → реальная отправка через API
