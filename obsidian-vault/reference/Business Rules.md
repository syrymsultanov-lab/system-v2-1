# Бизнес-правила (нарушать нельзя)

## Данные
1. **Contact ≠ Lead** — лиды только через форму лендинга, контакты = телефонная книга партнёра
2. Ручной перенос контактов в лиды **запрещён**
3. Max 20 таблиц в Supabase (сейчас 8)
4. Старый проект (66 таблиц, `ap-southeast-2`) — **мёртв**, не использовать

## Копирайтинг и compliance
5. **Никаких гарантий дохода** — требование InCruises + Meta/WABA
6. Scarcity tactics запрещены ("мест ограничено")
7. Прямое название MLM компании убрано из рекламного копирайтинга
8. Вся коммуникация должна быть Meta-safe
9. **AI-агент работает в полуавтомате** — LLM не отправляет сообщения лидам напрямую. Только генерирует черновик → партнёр редактирует/одобряет → отправка. Причина: WABA банит за нарушения Meta policy, LLM может непреднамеренно пообещать доход
9b. **Gate активации AI: обязательное обучение** — AI-агент НЕ активируется для партнёра, пока он не закончил обязательные модули обучения (минимум: «Старт партнёра», «Compliance: что нельзя обещать», «Дашборд SYSTEM V2», «AI-агент: работа с черновиками»). Это защита от compliance-нарушений: партнёр должен сначала выучить, что нельзя обещать гарантию дохода и какие формулировки банит Meta. Принято 2026-04-18 при заполнении вкладки `training.html`

## Язык
9. RU — primary
10. EN и KZ — как switchers

## Разработка
11. Один запрос = один выход. Не расширять scope без запроса
12. DB first — всегда проверять реальную схему перед генерацией кода
13. RLS gotcha: пустые результаты anon key → проверь политики, не данные
14. Lovable `types.ts` **не описывает реальную БД** — не использовать как источник
